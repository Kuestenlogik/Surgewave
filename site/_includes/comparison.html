Surgewave vs the field

Surgewave is built end-to-end on .NET 10 with a low-latency native transport, signed plugin ecosystem, AI pipeline nodes, and embedded mode — plus full Kafka-wire compatibility for migration. Here's where each tool lands.

Capability Surgewave Apache Kafka Redpanda Aeron
Setup & ops Single binary, no JVM, no ZooKeeper JVM cluster + KRaft / ZooKeeper Single binary (C++) Library, run in your process
Kafka wire compat 100 % (Kafka 4.x RPCs) Native 100 % None
Latency profile Low-latency native, Kafka-wire on par with Kafka JVM-bounded C++ broker, low-latency Microsecond UDP (library)
Runtime .NET 10 / C# 14, AOT-ready JVM (Scala/Java) C++ JVM
Plugin model Signed .swpkg packages, ALC isolation JAR plugins (Connect, MM2) Wasm transforms None
Embedded mode In-process broker (surgewave://embedded) No No Yes (it's a library)
Tiered storage Built-in (S3, Azure, GCS) 3.6+ (preview) Yes No

Surgewave fits .NET teams that want Kafka-wire compatibility, an embedded broker mode, and a signed plugin ecosystem — all in a single binary, no JVM. Public head-to-head benchmarks land with the 1.0 release; the table above summarises architecture, not measured numbers.
